Humidity Prediction Visualization Trust Study

9 Experimental Conditions - Between-Subjects Design

Study Overview

This study examines how different uncertainty visualization formats affect user trust and decision-making in Humidity predictions. Participants compare two cities (City A vs City B) using interactive visualizations and provide probability estimates, confidence ratings, and travel decisions.

For Researchers

Study Design: Two-phase between-subjects (No visualization → With visualization)

Data Collection: Automatic server-side saving to data/ directory

Version Tracking: Each URL corresponds to a specific experimental condition

Randomization: Send participants directly to specific URLs to assign conditions

Available Experimental Conditions

Version 1: Baseline

Shows only aggregated prediction lines without any uncertainty representation. Serves as the control condition for comparing the effectiveness of uncertainty visualizations.

Start Version 1 →

Version 2: PI Plot

Displays aggregated predictions with confidence bounds shown as shaded areas. Participants see uncertainty represented as regions with 95% probability coverage.

Start Version 2 →

Version 3: Ensemble Plot

Shows both aggregated predictions and multiple alternative prediction lines from different forecast agencies. Displays prediction variability explicitly.

Start Version 3 →

Version 4: Ensemble + Hover

Aggregated prediction by default, with alternative prediction lines revealed on hover. Tests progressive disclosure of uncertainty information.

Start Version 4 →

Version 5: PI Plot + Hover

Confidence bounds (PI plot) with hover functionality to reveal individual prediction scenarios. Combines static uncertainty with interactive exploration.

Start Version 5 →

Version 6: PI → Ensemble

Confidence bounds transform into ensemble plot on hover. Tests dynamic morphing between different uncertainty representation formats.

Start Version 6 →

Version 7: Buggy Control

Deliberately broken interactions including misaligned elements, wrong hover zones, and draggable components. Control condition for poor interface quality.

Start Version 7 →

Version 8: Bad Control

Poor UX patterns including forced clicks, timed pop-ups that disappear automatically, and unstable UI elements. Control for poor interaction design.

Start Version 8 →

Version 9: Combined PI + Ensemble

Shows both confidence bounds and alternative prediction lines simultaneously. Tests the effect of maximum uncertainty information display.

Study Duration: Approximately 15-20 minutes per participant
